## Week 1 — Foundation & CI Setup (Apr 14–18)

**Goal:** Everyone can run the project locally, CI is green on every push, and scaffolding is ready for feature work next week.

**Resources:** 2 BE + 3 FE (all 5)

### Backend (2 people)

| Task | Target | Notes |
|------|--------|-------|
| Monorepo setup | Tue–Wed | Python (FastAPI) backend + Next.js frontend. Use a simple workspace layout (e.g., `backend/` + `frontend/`). Shared API contract via OpenAPI schema auto-generated by FastAPI |
| FastAPI project scaffold + MySQL schema | Wed–Thu | All 4 tables: users, assets, repair_requests, repair_images. Use SQLAlchemy + Alembic for migrations. Include `version` column for optimistic locking from day 1 |
| Seed script with demo data | Thu–Fri | ~50 assets, 2 users per role, ~10 repair requests |
| CI pipeline: backend | Fri | Flake8 lint + mypy type-check + pytest (placeholder tests). GitHub Actions on every push |

### Frontend (3 people)

| Task | Target | Notes |
|------|--------|-------|
| Next.js project in monorepo | Tue–Wed | App Router, TypeScript strict mode |
| UI library setup (Ant Design / shadcn) | Wed | Pick one; establish component patterns |
| i18n framework (`next-intl`) | Wed–Thu | zh-TW + en. Set up early — retrofitting is painful |
| Layout: sidebar nav, header (no auth guard yet) | Thu–Fri | Role-aware menu structure prepared but not enforced until auth exists |
| CI pipeline: frontend | Fri | ESLint lint + TypeScript type-check. Runs on every push alongside backend CI |

### CI & Security Gates (shared effort)

| Task | Target | Notes |
|------|--------|-------|
| GitHub Actions workflow | Thu–Fri | Single workflow running FE lint/typecheck + BE lint/typecheck/test |
| gitleaks pre-commit hook | Thu | Secret scanning from day 1 (per `09-testing-strategy.md` §Secret Scanning) |
| Semgrep basic SAST | Fri | OWASP top-10 rule set in CI (per `09-testing-strategy.md` §SAST) |

### Milestone: `M1 — Skeleton Running`
- [ ] `docker compose up` or local dev scripts start both FE and BE
- [ ] DB schema deployed with seed data
- [ ] FastAPI auto-generated docs available at `/docs`
- [ ] CI pipeline runs lint + type-check + secret scan on every push
- [ ] No auth yet — that's Week 2

## Risk Register

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| FastAPI/SQLAlchemy ramp-up slows Week 1 | Medium | High | FastAPI has excellent docs + auto-generated Swagger. Seed script + schema by Thu W1 is the early warning |
| AWS setup takes longer than expected | Medium | Medium | Start Docker in W5 Mon. If AWS delays, demo on local Docker Compose |
| Integration bugs pile up in W3–W4 | High | Medium | FE/BE pair-test each API as it's built in W2–W3. Don't wait until W4 |
| Presentation spec released late | Medium | Low | Start slides with known rubric (architecture 25%, testing 25%). Adjust layout later |
| Live demo fails during presentation | Low | High | Record backup demo video in Buffer week. Have screenshots as fallback |
| Security CI gates too strict / slow | Low | Medium | Start with minimal gates in W1 (lint + gitleaks), expand progressively in W5 |

## Security CI Pipeline (Progressive Rollout)

Per `09-testing-strategy.md`, security gates are added progressively:

| Stage | When Added | Tools | Gate |
|-------|-----------|-------|------|
| Secrets | Week 1 | gitleaks (pre-commit + CI) | Block on any finding |
| SAST | Week 1 | Semgrep (OWASP top-10 rules) | Block on ERROR severity |
| Lint | Week 1 | ESLint (FE) + Flake8 (BE) | Zero errors |
| SCA | Week 5 | npm audit + pip-audit | Block on HIGH/CRITICAL CVE |
| Deep SCA | Week 5 | OWASP Dependency-Check | Block on CVSS ≥ 7 |
| Quality Gate | Week 5 | SonarQube | Quality Gate must pass |
